# 活动体系

## 常规活动

### 每周固定活动

#### 周六下午：迷你创客松 / AI共学活动
- **时长**：2-4小时
- **活动形式**：Demo + 共学 + 共创 + Social + 迷你黑客松
- **主题选定**：
  - 前一周投票确定当周主题
  - 可选主题：AI编程、游戏、视频、音乐、艺术、写作、教育、手搓硬件小产品等
- **活动要求**：
  - 以实操为主，必须带电脑
  - 现场可组队进行迷你黑客松
- **Demo/Pitch环节**：
  - 每周最多6个展示，每人5分钟
  - 完成度30%以上项目优先
  - 未被选上的可在交流环节进行展示
- **特色内容**：
  - 最新AI工具展示与实践
  - 活动全程记录并沉淀
  - 志愿者/赞助方可获得项目推广时间（限时）

#### 周六晚上：AI酒馆（每月两次）
- 嘉宾分享
- AI热点讨论
- 轻松社交氛围
- 社区成员交流与组队
- Party Time / High跳环节

### 每月固定活动

#### 第一个周日下午：Demo Day
- 项目展示与分享
- 社区月度总结（公开会议）
- 下月活动规划
- 优秀项目评选

#### Build In Public（每月1日起，持续21天）
- 开发者公开分享项目进度
- 获取第一批用户反馈
- 获得社区内鼓励
- 项目进展将在社区媒体平台展示，帮助开发者获得流量

### 季度活动
- 黑客松大赛（每季度最后一个周末）
  - 未来可能发展为俱乐部内赛+小区赛+大区赛（类似ToastMaster）
- 大型社区聚会
- Hackerhouse活动

## 活动执行标准

### 活动前准备

#### 时间节点
- **T-7天**
  - 发布活动预告
  - 确定主题（共学活动需投票）
  - 准备活动文案和海报
- **T-3天**
  - 场地确认
  - 设备检查
  - 参与者提醒
- **T-1天**
  - 物料准备就绪
  - 最终确认名单

#### 准备清单
- **必备物料**
  - 活动海报
  - 签到表
  - 投影设备
  - 网络设备
  - 摄影设备
- **可选物料**
  - 茶歇
  - 名片墙
  - 活动手册

### 活动执行

#### 现场管理
- 签到登记（收集联系方式）
- 拍照记录
- 直播/录像（可选）
- 项目记录表

#### 标准流程
1. **开场介绍**（15分钟）
   - 社区简介
   - 活动流程
   - 参与者自我介绍
2. **主题活动**（90-120分钟）
3. **互动环节**（30分钟）
4. **项目展示**（30分钟）
5. **总结反馈**（15分钟）

### 活动后处理

#### 24小时内完成
- 活动总结
- 照片整理
- 参与者反馈收集
- 项目信息整理

#### 48小时内完成
- 更新社区网站
- 发布活动回顾
- 整理活动资料
- 更新项目展示页面
