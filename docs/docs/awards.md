# 社区称号与荣誉制度

## 一、称号体系概览

我们的称号体系包含全局称号和分部称号，旨在激励和表彰为社区发展做出贡献的成员，同时支持各城市分部和俱乐部的独立发展。

### 1. 全局基础称号

#### 创始荣誉称号
- **创始发起人（Founding Contributor）**
  - **定义**：社区的最初创建者和开拓者
  - **人数限制**：3-5人
  - **授予条件**：
    - 参与社区的创立和早期发展
    - 为社区的基础建设做出重要贡献
  - **权益**：
    - 社区重大决策的投票权
    - 在社区历史中永久记录
    - 获得专属徽章和证书

#### 运营核心称号
- **核心组织者（Core Organizer）**
  - **定义**：社区运营和发展的核心推动者
  - **授予条件**：
    - 连续3个月主持社区活动
    - 推动2个及以上社区项目落地
    - 累计获得50积分
  - **权益**：
    - 社区项目资源优先调配权
    - 重要决策的参与权
    - 获得专属徽章

### 2. 分部称号体系

#### 城市分部称号
- **分部发起人（Chapter Founder）**
  - **定义**：城市分部或俱乐部的创建者
  - **人数限制**：每个分部1-3人
  - **授予条件**：
    - 成功建立并运营城市分部/俱乐部
    - 持续3个月以上的活动组织
  - **权益**：
    - 分部重要决策的投票权
    - 获得分部发起人徽章
    - 优先使用分部资源

- **分部核心（Chapter Core）**
  - **定义**：分部的主要运营者
  - **授予条件**：
    - 连续3个月参与分部运营
    - 累计30积分
  - **权益**：
    - 分部活动的组织权
    - 分部资源调配建议权

- **分部活跃者（Chapter Active）**
  - **定义**：分部的活跃参与者
  - **授予条件**：
    - 季度内参与4次以上分部活动
    - 累计15积分
  - **权益**：
    - 分部活动优先参与权
    - 获得活跃者徽章

### 3. 成就称号（可叠加获得）

#### 技术创新相关
- **创新先锋（Innovation Pioneer）**
  - **授予条件**：
    - 在周周黑客松中完成3个以上创新项目
    - 项目获得社区评选奖项
  - **权益**：
    - 优先获得项目孵化支持
    - 技术分享优先展示机会

- **开源贡献者（Open Source Contributor）**
  - **授予条件**：
    - 为社区开源项目贡献代码
    - 维护或参与社区技术基础设施建设
  - **权益**：
    - 获得开源贡献者徽章
    - 技术项目优先支持

#### 社区建设相关
- **知识传播者（Knowledge Sharer）**
  - **授予条件**：
    - 发表3篇以上高质量技术文章
    - 主持2次以上技术分享会
  - **权益**：
    - 获得知识贡献徽章
    - 社区资源优先使用权

- **社群引导者（Community Guide）**
  - **授予条件**：
    - 帮助10名以上新成员融入社区
    - 组织3次以上社群活动
  - **权益**：
    - 获得社群引导者徽章
    - 社区活动优先参与权

### 4. 动态称号

- **黑客松明星（Hackathon Star）**
  - **更新周期**：每月
  - **范围**：全局/分部
  - **授予条件**：
    - 当月黑客松项目获得最高评分
    - 项目具有创新性和实用价值

- **活跃贡献者（Active Contributor）**
  - **更新周期**：每季度
  - **范围**：全局/分部
  - **授予条件**：
    - 季度内参与4次以上活动
    - 累计获得20积分

## 二、积分机制

### 1. 积分获取方式

| 贡献类型 | 积分 | 说明 |
|---------|------|------|
| 项目开发 | 15分/个 | 完成一个周周黑客松项目 |
| 技术分享 | 10分/次 | 举办技术分享会或工作坊 |
| 文章撰写 | 8分/篇 | 发表技术文章或项目总结 |
| 活动参与 | 3分/次 | 参与社区组织的各类活动 |
| 新人引导 | 5分/人 | 帮助新成员融入社区 |
| 开源贡献 | 8分/次 | 为社区项目提交代码或文档 |
| 跨部协作 | 10分/次 | 参与跨分部的项目或活动 |

### 2. 积分使用

积分可用于：
- 兑换社区专属周边
- 获取技术课程优惠
- 申请项目孵化支持
- 达到称号授予条件
- 获得活动优先权

## 三、称号授予与管理

### 1. 授予流程
- 全局基础称号：由社区委员会提名并投票决定
- 分部称号：由分部管理团队决定
- 成就称号：达到条件后自动授予
- 动态称号：系统自动根据数据更新

### 2. 分部管理自治
- 各分部可在全局框架下自主管理本分部称号
- 分部可设立特色称号，需报备社区委员会
- 分部可根据本地特点设置额外的积分奖励规则

### 3. 称号展示
- 社区官网个人主页展示
- 分部页面特别展示
- 活动优先级和权益体现
- 年度社区报告中的贡献者表彰

### 4. 称号退出机制
- 动态称号：未达到条件自动退出
- 分部称号：由分部管理团队定期审核
- 其他称号：严重违反社区规范可被取消
